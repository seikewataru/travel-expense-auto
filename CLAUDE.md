# travel-expense-auto プロジェクトルール

## 概要
交通費関連の費用明細取得〜MF会計Plus仕訳自動計上〜稟議突合振替〜部門別ROI可視化まで一気通貫で行うWebアプリ。
旧auto-mf-journalプロジェクトを2026-03-18に統合。

## 要件定義
詳細は [PRD.md](./PRD.md) を参照。

## 技術スタック
- **言語**: Python 3.11+
- **UI**: Streamlit（Webアプリ化、チーム利用）
- **可視化**: Plotly / Altair（ROIダッシュボード）
- **MF経費**: requests（公式REST API、OAuth 2.0）
- **EXカード・Racco・じゃらん・タイムズカー**: Playwright（管理画面スクレイピング→CSV自動DL）
- **Google Sheets**: gspread（部署マスタ・売上データ・仕訳出力先）
- **認証情報**: `.env`ファイル（dotenv）
- **GCP認証**: サービスアカウント `~/.config/gcp/service-account.json`（ir-db-automationと共有）
- **デプロイ**: Streamlit Cloud（パスワード保護）

## 既知の差異（経理集計シートとの突合結果、許容済み）

### 税抜計算の端数差（±数百円）
- ツールは一律 `round(税込額 / 1.1)` で税抜変換している
- 経理側は明細の税区分（課税仕入10% / 経過措置80% / 対象外）を個別に処理
- **経過措置80%**: 適格請求書でないホテル領収書 → 税額計算が異なる（例: 森久由生 ¥197差）
- **宿泊税（対象外）**: ホテル代に含まれる宿泊税は消費税対象外 → 分離計算が必要だが未対応（例: 澤田和哉 ¥14差）
- **対応方針**: 許容する（修正コスト > 差額。2026-03-10 決定）

### 稟議申請者 vs 経費申請者の不一致
- ツール（②）はMF経費APIの `office_member`（経費申請者）で集計
- 経理集計シート（①）は事前申請番号経由の稟議申請者で集計
- 同一経費が別人に帰属するケースがある（例: MF明細41214 → ①は梅田力、②は近藤健太）
- **対応方針**: ②の経費申請者ベースで集計する（APIから取得できる情報の制約）
